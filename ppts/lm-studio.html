⌨️ 快速加载模型
在任意界面按 Cmd+L（Mac）或 Ctrl+L（Win/Linux）可弹出模型选择器，快速切换已下载模型。
💡 System Prompt 设置
点击顶部 System Prompt 按钮输入角色指令，如"你是一名专业的 Python 工程师，回答简洁"，每次对话自动携带。
⚠️ 参数调整无需重载
Chat 右侧面板中的 Temperature、Context Length 等参数修改后，下一条消息立即生效，无需重新加载模型。但修改 GPU 层数、Context 上限等加载参数则需重载。

📌 关键配置
  • CORS：开启后允许浏览器前端应用跨域请求
  • 本地网络：开启后局域网内其他设备可访问（地址变为 0.0.0.0:1234）
  • 端口：可在 Settings 中修改，避免与其他服务冲突
💡 兼容工具
Cherry Studio · Open WebUI · Cursor · Continue(VSCode) · Obsidian · 任何支持自定义 OpenAI endpoint 的应用，填入 http://localhost:1234/v1 即可
Python · openai 库调用
from openai import OpenAI
 
# 指向本地 LM Studio
client = OpenAI(
  base_url="http://localhost:1234/v1",
  api_key="lm-studio" # 任意值
)
resp = client.chat.completions.create(
  model="local-model",
  messages=[{"role":"user",
            "content":"你好"}]
)
print(resp.choices[0].message.content)

🔗 LM Link 工作原理
  • 基于 Tailscale 加密 VPN Mesh，设备间直连，不经过 LM Studio 服务器
  • 对话内容、推理过程完全在你的设备间传输
  • 远程模型在 Chat / API 中以本地方式显示，工具无感知
📌 免费限额
目前 LM Link 免费版支持最多 2 个用户，每人 5 台设备（共 10 台）。与现有 Tailscale VPN 共存，互不干扰。
1
登录 LM Studio Hub
在 GUI 点击 LM Link 标签 → Sign in，或命令行 lms login
2
添加远程设备
点击 Add a remote machine，在目标机器上执行 lms link enable
3
加载远程模型即用
回到 Chat，模型选择器中远程设备的模型自动出现，选中即可，API 仍用 localhost:1234

🛠️ CLI 常用命令
lms CLI
# 查看已下载模型
$ lms ls
# 下载模型（含量化选择）
$ lms get llama-3.2-8b@q4_k_m
# 加载并开启 server
$ lms load qwen2.5-7b
$ lms server start
# LM Link 操作
$ lms login
$ lms link enable
🔧 My Models 配置技巧
设置模型默认参数
进入 My Models 标签，点击模型右侧 ⚙️ 图标，可为每个模型单独配置默认 Temperature、Context 等，下次加载自动应用。
固定常用模型
在 My Models 中右键模型，选 Pin to top，常用模型置顶，避免在长列表中查找。
外部 GGUF 文件导入
已有 .gguf 文件？在 My Models 点 Import Model，或直接将文件放入 ~/.lmstudio/models/ 目录，重启后自动识别。
⚠️ 常见问题速查
  • 生成极慢：Settings → GPU → 确认 GPU offload 层数 > 0
  • 加载失败：内存不足，换 Q3/Q4 更低量化版本
  • 中文乱码：换 Qwen 系列或 DeepSeek 中文模型
  • API 拒绝连接：Local Server 未 Start，或端口被占用
  • LM Link 无法连接：两台设备需同时登录同一 Hub 账号
