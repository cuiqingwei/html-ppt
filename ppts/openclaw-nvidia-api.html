Chapter 02 · 注册 & 获取 Key

四步获取 NVIDIA API Key

1
访问 NVIDIA Build 平台
打开 build.nvidia.com，点击右上角「Sign In」→「Create Account」注册账号（支持邮箱注册，无需信用卡）。
2
验证手机号码
注册完成后务必验证手机号，否则无法获取 API 调用权限。在手机号码处直接填写 +86 开头的国内号码，点击「Send Code via SMS」接收验证码。
3
创建 API Key
验证完成后，点击右上角头像 → 选择「API Keys」→「Generate Key」。密钥格式为 nvapi-xxxxxxxxxxxxxxxxxxxxxxxx，生成后立即复制保存，页面关闭后无法再次查看。
4
确认模型访问权限
在平台搜索 glm 或 minimax，进入模型页面点击「View Code」可看到示例代码，确认 API 地址为 https://integrate.api.nvidia.com/v1。
💡 提示
GLM-4.7 和 MiniMax M2.1 目前可能尚未在官方模型广场完整列出，但 API 实测可正常调用。API Key 申请后即可直接使用这两个模型。

Chapter 03 · 配置 openclaw.json

找到并修改配置文件

配置文件位于 ~/.openclaw/openclaw.json（旧版为 ~/.clawdbot/clawdbot.json）

找到配置文件
# 查找配置文件位置
$ ls ~/.openclaw/
openclaw.json logs/ cache/

# 用编辑器打开（任选其一）
$ nano ~/.openclaw/openclaw.json
$ code ~/.openclaw/openclaw.json  # VS Code
$ vim ~/.openclaw/openclaw.json
⚠️ 注意版本差异
OpenClaw 经历了 Clawdbot → Moltbot → OpenClaw 的更名。新版（2026.1.24+）内置配置向导，可用 openclaw onboard 自动完成配置。旧版需手动编辑 JSON 文件。
Chapter 03 · 配置 openclaw.json

插入 NVIDIA Provider 配置

在 "providers": {"{}"} 后插入以下内容（GLM-4.7 + MiniMax M2.1 双模型）

"nvidia": {
  "baseUrl": "https://integrate.api.nvidia.com/v1",
  "apiKey": "nvapi-你申请到的Key",
  "api": "openai-completions",
  "models": [
    {
      "id": "z-ai/glm4.7",
      "name": "GLM-4.7",
      "reasoning": false,
      "input": ["text"],
      "cost": { "input": 0, "output": 0, "cacheRead": 0, "cacheWrite": 0 },
      "contextWindow": 128000,
      "maxTokens": 8192
    },
    {
      "id": "minimaxai/minimax-m2.1",
      "name": "MiniMax-M2.1",
      "reasoning": false,
      "input": ["text"],
      "cost": { "input": 0, "output": 0, "cacheRead": 0, "cacheWrite": 0 },
      "contextWindow": 128000,
      "maxTokens": 8192
    }
  ]
}
关键字段说明
"api": "openai-completions" 指定使用 OpenAI 兼容接口，cost 字段全部填 0 表示免费，避免 OpenClaw 显示错误费用估算。

Chapter 05 · 进阶配置

方案 A vs 方案 B

根据需求选择最适合的接入方式。

🔗
方案 A · NVIDIA NIM 中转（推荐入门）
✅ 一个 Key 调用多个模型
✅ OpenAI 兼容，配置简单
✅ 完全免费
⚠️ 模型可能有隐性限流
接口：integrate.api.nvidia.com/v1
🇨🇳
方案 B · 国内直连原生 API（推荐稳定）
✅ 智谱/MiniMax 官方接口，更稳定
✅ Anthropic 兼容格式，功能更完整
✅ 支持思考模式（reasoning）
⚠️ 需分别注册两家平台获取 Key
GLM：open.bigmodel.cn/api/anthropic
💡 方案 B 的 GLM-4.7 配置片段
将 "api" 改为 "anthropic-messages"，"baseUrl" 改为 "https://open.bigmodel.cn/api/anthropic"，"reasoning": true 开启思维链模式。
Chapter 05 · 进阶配置

多模型备选 & 故障转移

配置主模型 + 备选链，让 OpenClaw 在模型不可用时自动切换。

// 在 agents.defaults 中配置主备模型
"agents": {
  "defaults": {
    "model": {
      "primary": "nvidia/z-ai/glm4.7",
      "fallback": "nvidia/minimaxai/minimax-m2.1"
    }
  }
}

// 也可以配置更多备选（适合生产环境）
"model": {
  "primary": "nvidia/z-ai/glm4.7",
  "fallbacks": [
    "nvidia/minimaxai/minimax-m2.1",
    "anthropic/claude-sonnet-4-6"  // 付费兜底
  ]
}
限流自动退避机制
当某模型触发频率限制时，OpenClaw 会自动标记冷却期并切换到备选模型，使用指数退避策略（计费类失败：初始 5 小时，翻倍递增，上限 24 小时）。

Chapter 06 · 故障排查

错误速查 & 解决方案

错误现象原因解决方案
HTTP 401 Unauthorized API Key 填写错误或未生效 确认 Key 格式 nvapi-xxx，重启网关 openclaw gateway --force
模型不存在 / 404 model id 填写错误 GLM：z-ai/glm4.7，MiniMax：minimaxai/minimax-m2.1，注意前缀
配置修改无效 网关未重启 执行 openclaw gateway --force 强制重启
速率限制 / 429 免费配额耗尽或请求过频 配置备选模型，或等待冷却后重试
JSON 解析错误 配置文件格式不合法 用 cat openclaw.json | python3 -m json.tool 验证 JSON 格式
接口格式错误 api 类型填错（NVIDIA 是 OpenAI 格式） 确保 "api": "openai-completions"，不要填 anthropic-messages
⚠️ 踩坑提醒
NVIDIA NIM 接口是 OpenAI 兼容格式，不是 Anthropic 格式。如果你混用了两种格式的配置，会导致认证通过但响应解析失败。方案 B 的智谱/MiniMax 官方接口才支持 Anthropic 格式。
